🛠️ 工业硬核驱动：个人博客全系统设计规范 (v1.0)
一、 设计哲学：机械与信息的融合
- 核心理念：将博客视作一台“正在运行的精密机器”。每一篇文章是一个“模块”，每一个标签是一个“接口”，评论区则是“通讯协议”。
- 视觉特征：去装饰化、功能导向、高对比度、网格对齐。

三、 UI 布局架构 (Global Layout)
1. 顶部导航栏 (Signal Header)
- 样式：半透明磨砂质感 (backdrop-filter: blur(12px))。
- 细节：底部有一条 1px 的暗金细线。左侧 Logo 使用等宽字体（Monospace），模拟代码行；右侧导航项在 Hover 时显示类似“选中开关”的矩形边框。
- 组件：[ HOME ] [ ARCHIVE ] [ PROJECTS ] [ STATUS: ONLINE ]
2. 侧边栏 (Modular Sidebar)
- 定位：存放个人信息、标签云、运行日志。
- 设计：每个模块使用类似“抽屉”的凹陷效果。
- 特色模块 - 运行监控：显示博客的实时数据（如：文章总数、访客计数、服务器响应时间），使用绿色点阵字体模拟 80 年代显示器效果。
3. 内容主体 (Main Console)
- 文章列表卡片：
  - - 左侧带有一个垂直的“状态条”，不同类别的文章对应不同颜色的状态条（技术：蓝色；生活：绿色；思考：黄色）。
  - - 卡片右下角标注“十六进制 ID”或“时间戳”，增加科技感。
- 排版：正文字体推荐使用 Inter 或 Source Code Pro，行高设定为 1.7 以保证阅读舒适度。
四、 核心组件设计规范 (Component Specifications)
1. 按钮 (Actuators)
- 形态：微小圆角 (2px)。
- 阴影：使用 box-shadow: 4px 4px 0px #000 营造硬朗的立体感。
- 交互：点击时 transform: translate(2px, 2px)，阴影消失，模拟物理按键被按下的过程。
2. 代码块 (Logic Core)
- 外观：深黑色底色，窗口左上角模拟 macOS 或终端的“红黄绿”三个小圆点，但颜色做暗色处理。
- 字体：JetBrains Mono 或 Fira Code。
- 边框：右侧添加一个细长的“滚动条槽”，模拟工业控制屏。
3. 标签 (Tags/Markers)
- 设计：外观像“电缆标签”或“撕裂式胶带”。
- 样式：背景色为暗灰色，文字为亮色，左侧带有一个 # 符号。
五、 交互动效 (Machine Motion)
- 加载动画 (Loading)：不要转圈圈，改用从 0% 到 100% 的数字跳动，或者类似 CMD 窗口的光标闪烁。
- 页面切换：采用“扫描线”效果或简单的“淡入+向上位移”，模拟屏幕刷新的感觉。
- Hover 状态：所有可点击元素在鼠标悬停时，增加一个极短的 0.1s 的发光（Glow）过渡。
六、 响应式适配规范 (Responsive Grid)
1. PC 端 (Ultra-Wide)：三栏布局（导航 | 内容 | 侧边栏）。
2. 笔记本 (Laptop)：两栏布局（内容 | 侧边栏），导航栏置顶。
3. 移动端 (Mobile)：单栏流式布局。
- 侧边栏折叠进底部抽屉或汉堡菜单。
- 卡片间距加大，方便触控。
- 按钮高度至少保持 44px，确保“工业操作”的精准度。
七、 页面装饰元素 (Deco Elements)
- 网格背景：背景中使用极其微弱的 16px * 16px 网格点或线。
- 角落标记：在页面四个角添加类似 [ + ] 或 L 型的直角边框。
- 字体装饰：在页面不显眼处放置一些无意义的序列号（如：VER: 2.0.4-STABLE），强化系统感。
